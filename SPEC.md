# 訪問マップ アプリ仕様

## 概要

複数のユーザーが、日本地図上の都道府県に「訪問状態」を記録・共有するWebアプリ。  
データはブラウザの localStorage に保存し、サーバー不要で動作する。

---

## 画面構成

```text
UserList（ユーザー管理画面）
    ↓ 「マップを開く」
VisitedMap（訪問マップ画面）
```

---

## 画面仕様

### UserList（ユーザー管理画面）

起動時の初期画面。ユーザーの一覧表示とCRUD操作、訪問状態の管理を行う。

#### 表示内容

- ユーザー一覧テーブル（名前列）
- ユーザー操作ボタン：追加 / 編集 / 削除
- 訪問状態一覧テーブル（状態名・色・並び順）
- 訪問状態操作ボタン：追加 / 編集 / 削除 / 順序変更
- 「マップを開く」ボタン（ユーザーが1人以上いる場合のみ有効）
- エクスポート / インポートボタン

#### ユーザー操作

| 操作         | 動作                                            |
| ------------ | ----------------------------------------------- |
| 行クリック   | ユーザーを選択／解除（複数選択可）              |
| 追加         | 「ユーザーN」という名前で新規作成               |
| 編集         | 1件選択時のみ有効。名前を編集するモーダルを開く |
| 削除         | 選択中のユーザーを全て削除                      |
| マップを開く | VisitedMap画面へ遷移（ユーザー0人時は無効）     |

#### ユーザー操作 — 編集モーダル

- タイトル：「ユーザー編集」
- 入力項目：名前（テキスト）
- ボタン：保存 / 閉じる

#### 訪問状態操作

| 操作       | 動作                                                |
| ---------- | --------------------------------------------------- |
| 行クリック | 訪問状態を選択／解除（複数選択可）                  |
| 追加       | 新しい訪問状態を作成するモーダルを開く              |
| 編集       | 1件選択時のみ有効。訪問状態を編集するモーダルを開く |
| 削除       | 選択中の訪問状態を全て削除（※後述）                 |
| ↑ / ↓      | 1件選択時のみ有効。表示順を1つ上下に移動            |

**訪問状態を削除したとき：** その状態が設定されていた全ユーザーの都道府県記録も合わせて削除する。

#### 訪問状態編集モーダル

- タイトル：「訪問状態編集」（追加時は「訪問状態追加」）
- 入力項目：
  - 名前（テキスト）
  - 色（カラーピッカー）
- ボタン：保存 / 閉じる

---

### VisitedMap（訪問マップ画面）

日本地図を表示し、都道府県ごとに訪問状態を記録・比較する画面。

#### マップ表示内容

- 日本地図SVG
  - 768px以上：`map-full.svg`
  - 768px未満：`map-mobile.svg`
- 都道府県は選択中ユーザーの訪問状態に応じた色で塗りつぶして表示（未設定は白）
- **比較モード**（ユーザーを2人選択時）：都道府県を左右に分割し、左に1人目・右に2人目の色を表示。比較モード中は訪問状態の編集不可
- ユーザー選択UI（最大2人まで選択可）。画面遷移直後は誰も選択されていない状態で始まる
- 訪問状態の凡例（状態名と色）

#### マップ操作

| 操作                           | 動作                                                                   |
| ------------------------------ | ---------------------------------------------------------------------- |
| 都道府県クリック               | 訪問状態をサイクル変更（未設定 → 状態1 → 状態2 → … → 未設定）          |
| 都道府県クリック（比較モード） | 変更不可。クリックした県の全ユーザーの訪問状態をポップアップで一覧表示 |
| ユーザー選択                   | 表示するユーザーを切り替え（最大2人まで選択可）                        |
| UserListへ戻る                 | ユーザー管理画面へ戻る                                                 |

サイクルの順序は `areaStatusModels` の配列順。訪問状態が0件の場合はクリックしても「未設定」のまま変化しない。

## データ構造

localStorage のキー `saveData` に JSON で保存。

```ts
SaveDataModel {
  version:          number             // データフォーマットのバージョン（現在: 1）
  areaStatusModels: AreaStatusModel[]  // 訪問状態の定義（配列順が表示順）
  userDataModels:   UserDataModel[]    // ユーザーと訪問記録
}

AreaStatusModel {
  areaStatusUuid:  string  // 状態ID
  areaStatusName:  string  // 状態名
  areaStatusColor: string  // 表示色（CSS カラー値 例: '#81C784'）
}

UserDataModel {
  userUuid:       string          // ユーザーID
  userName:       string          // ユーザー名
  mapDataModels:  MapDataModel[]  // 都道府県ごとの訪問記録
}

MapDataModel {
  areaId:         string  // 都道府県コード（JIS X 0401準拠、SVGのdata-code属性と対応。例: '13' = 東京都）
  areaStatusUuid: string  // どの訪問状態か
}
```

## データのエクスポート / インポート

UserList画面に「エクスポート」「インポート」ボタンを設置する。

| 操作         | 動作                                                              |
| ------------ | ----------------------------------------------------------------- |
| エクスポート | `saveData` の JSON をファイルとしてダウンロード                   |
| インポート   | JSON ファイルを選択し、現在のデータを上書き（確認ダイアログあり） |

**インポート時のバリデーション：**

- `version` がアプリの対応バージョンと一致しない場合はエラーメッセージを表示し、上書きしない
- JSON の構造が不正な場合も同様にエラーを表示し、上書きしない

---

## 初期データ

初回起動時（localStorage が空）は以下の状態で始まる。

- ユーザー：なし
- 訪問状態：以下の3つをデフォルトで設定（この順序で表示）

| 状態名 | 色              |
| ------ | --------------- |
| 通過   | `#FFD54F`（黄） |
| 観光   | `#81C784`（緑） |
| 居住   | `#64B5F6`（青） |
